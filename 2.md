### ✅ c++
namespace: 서로 다른 namespace를 사용함으로써 같은 이름의 여러 함수를 사용할 수 있음  
타입변환: float(a), int(a) (C: (float)a, (int)a)  
C: 모든 함수, 변수 이름이 달라야함  
C++: 구분만 된다면 동일한 이름 사용 가능  
template: 데이터 타입에 의존하지 않고 함수나 클래스를 작성할 수 있게 해주는 도구  
```cpp
#include <iostream>
using namespace std;

template <typename T>
T Max (T a, T b)
{ // Maximum of two values of type T
    return (a > b) ? a : b;
}

int main()
{ // Single function using template
    int i1 = 2, i2 = 3;
    cout << "Max(i1, i2) = " << Max(i1, i2);

    double f1 = 5.8, f2 = 7.2;
    cout << "; Max(f1, f2) = " << Max(f1, f2);

    char c1 = 'B', c2 = 'C';
    cout << "; Max(c1, c2) = " << Max(c1, c2);

    return 0;
}
```
메모리 동적 할당(new): `p1 = new int`, `p2 = new int[num]`  
메모리 삭제(delete): `delete p1`, `delete[] p2`  
reference: 똑같은데 다른 이름을 가지는거  
```cpp
#include <iostream> 
void Increment(int& n) { n++; }
void Print(int n) { std::cout << "Value = " << n; }
int main()
{
    int count = 0;
    Increment(count); 
    Print(count);
    return 0;
}
```
class: struct + 함수  
객체지향: 캡슐화, 추상화, 상속, 다용성  
constructor: 객체 생성  
destructor: 객체 삭제, 메모리 해제  
new로 동적할당을 하면 destructor로 delete를 해주어야 메모리 누수를 방지할 수 있음  
virtual function: 부모클래스에서 선언되어 자식 클래스에서 재정의 될 것이 기대되는 함수, 동적바인딩(실행시점에 호출될 함수 결정)  
abstract class: 동일한 틀을 만들어 놓고 사용하기 위해서 그 자체로는 사용하지 않는 추상화된 클래스  
Stack: LIFO  
Queue: FIFO  
Deque: 앞뒤 다 push, pop 가능  
List: 순서가 있는 값  
Linked List: 포인터로 연결됨
